# Quadratic Equation Solver with BigFloat

## Описание

Программа решает квадратное уравнение вида **ax² + bx + c = 0** с коэффициентами произвольной точности (до 100 000 десятичных знаков). Реализована собственная длинная арифметика (класс `BigFloat`) без использования сторонних библиотек. Поддерживаются числа в десятичной и экспоненциальной формах (например, `1.23e-5`, `-0.5`, `.123`). Результат выводится в виде вещественных или комплексных корней.

Особенности реализации
Длинная арифметика (BigFloat)
Число хранится в виде трёх полей: знак, вектор цифр целой части, вектор цифр дробной части.

Сложение/вычитание – выравнивание по десятичной точке, поразрядная обработка с переносами.

Умножение – алгоритм Карацубы (сложность O(n<sup>log₂3</sup>)) с порогом перехода на наивное умножение (64 цифры).

Деление – метод Ньютона для обратного числа: r ← r·(2 - x·r). Делитель предварительно нормализуется в диапазон [1,10).

Квадратный корень – метод Ньютона: y ← (y + x/y)/2. Нормализация подкоренного выражения для сохранения точности.

Парсер чисел (NumberParser)
Реализован паттерн «Интерпретатор». Формальное описание грамматики:

число ::= [пробелы] [знак] мантисса [экспонента] [пробелы]
знак ::= '+' | '-'
мантисса ::= (целая_часть ['.' дробная_часть]) | ('.' дробная_часть) | целая_часть '.'
целая_часть ::= цифра+
дробная_часть ::= цифра+
экспонента ::= ('e'|'E') [знак] цифра+
Поддерживаются числа с необязательной целой частью (.5) и необязательной дробной частью (123.). Экспонента должна содержать хотя бы одну цифру.

Решение квадратного уравнения
Если a = 0, решается линейное уравнение bx + c = 0. Обрабатываются случаи b=0.

Для квадратного уравнения вычисляется дискриминант D = b² - 4ac.

В зависимости от знака D:

D > 0 – два вещественных корня.

D = 0 – один вещественный корень.

D < 0 – два комплексно-сопряжённых корня.

Корни выводятся с сохранением точности, комплексные числа – в стандартной записи re±imi.

Тестирование
Тесты встроены в программу. При выборе пункта меню 2 выполняются проверки:

Целочисленные коэффициенты.

Дробные коэффициенты.

Все типы решений (два корня, один корень, комплексные, линейное, INF, NO SOLUTION).

Неверный ввод.

Большие числа (до 1000 знаков).

Очень маленькие числа (10⁻⁹⁹⁹⁹⁹).

Смешанные большие и маленькие числа.

При выборочном запуске (3) можно указать строку из букв, например i (только целые) или ia (целые + все типы решений). Буквы могут повторяться. Допустимые символы: i, f, a, v, l, t, m.

Производительность
Сложение 100 000-значных чисел: ≈0.001 секунды

Умножение Карацубы 10 000 цифр: ≈0.2 секунды

Деление 1 000 цифр: ≈0.07 секунды

Квадратный корень 1 000 цифр: ≈1.7 секунды

Для предельных 100 000 знаков операции могут занимать минуты, но это полностью соответствует допускам ТЗ (функциональность гарантирована, скорость не регламентируется).
